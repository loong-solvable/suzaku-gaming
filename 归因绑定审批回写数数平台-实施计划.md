## 1. 需求概述

管理员通过一个归因绑定申请后，将该角色的 `accountId`、`roleId` 和 CPS 分组写入数数平台（ThinkingData）的 CPS 维度表，使 CPS 同步服务能正确识别该用户。

**当前缺失环节：** 审批通过仅更新本地 PostgreSQL（`roles.cps_visible=true`），未同步到数数平台维度表，导致 CPS 同步任务 LEFT JOIN 该用户时结果为 NULL。

| 尝试值 | 错误信息 | 结论 |
|--------|---------|------|
| `datatableName=ta_dim.dim_22_0_518509` | 数据表不存在或已删除 | SQL 查询用的维度表内部名，API 不识别 |

## 3. 数数平台 CPS 数据表结构

**数据表名称：** `cps_group`

| 列名（API/CSV 用） | SQL 查询时的名称 | 类型 | 说明 |
|-------------------|-----------------|------|------|
| `account_id` | `#account_id@account_id` | varchar | **主键**（首列） |
| `role_id` | `#account_id@role_id` | varchar | 角色ID |
| `cps_group` | `#account_id@cps_group` | varchar | CPS 分组 |

> 注：SQL 查询（如 `cps-sync.service.ts` 中的 LEFT JOIN）使用的是带 `#account_id@` 前缀的列名，但数据表 API 上传 CSV 时使用**不带前缀的原始列名**。

## 4. 方案设计

### 4.1 整体架构

```
管理员审批通过
  │
  ├── [事务内] 更新本地数据库（已有逻辑，不变）
  │     ├── binding_applies.status → approved
  │     ├── roles.cps_visible → true
  │     └── audit_log 写入
  │
  └── [事务后] fire-and-forget 调用回写（新增）
        │
        ▼
  TaDatatableService.writeCpsDim(roleId, cpsGroup)  ← 完全新增的独立服务
        │
        ├── 查 roles 表获取 accountId
        ├── 生成内存 CSV（account_id, role_id, cps_group 三列）
        ├── POST /open/datatable/updateDatatable (INCR_UPDATE)
        │
        ├── 成功 → logger.log
        └── 失败 → logger.error（不影响审批结果）
```

### 4.2 设计原则

| 原则 | 说明 |
|------|------|
| 本地审批优先 | 回写放在事务之后。即使数数平台不可达，审批结果不受影响 |
| 新增代码为主 | 核心逻辑放在**全新文件** `ta-datatable.service.ts`，已有文件只做最小改动 |
| fire-and-forget | 回写调用不阻塞 HTTP 响应返回，通过 `.catch()` 兜底避免未处理异常 |
| 独立可测 | 新服务自包含 Logger/Config/Prisma，可独立单元测试 |

### 4.3 API 调用详情

| 项目 | 值 |
|------|-----|
| URL | `https://ss-search.tapplus.com:18993/open/datatable/updateDatatable` |
| 方法 | POST |
| Content-Type | multipart/form-data |

**请求参数：**

| 参数 | 位置 | 值 |
|------|------|-----|
| `token` | query | 环境变量 `TA_PROJECT_TOKEN` |
| `projectId` | query | `22`（环境变量 `TA_PROJECT_ID`） |
| `datatableName` | query | `cps_group`（环境变量 `TA_DATATABLE_NAME`） |
| `updateType` | query | `INCR_UPDATE` |
| `file` | body | 动态生成的 CSV 文件（3 列） |

**CSV 文件内容示例：**

```csv
account_id,role_id,cps_group
1234567890,role_abc_123,WA_CPS_link_xxx
```

### 4.4 数据映射

| 来源 | → | CSV 列 | 说明 |
|------|---|--------|------|
| `role.accountId`（通过 `apply.roleId` 查本地 `roles` 表） | → | `account_id` | 主键 |
| `apply.roleId` | → | `role_id` | 角色ID |
| `apply.platform` | → | `cps_group` | CPS 分组标识 |

### 5.2 新增文件：`ta-datatable.service.ts`（核心）

路径：`src/modules/thinkingdata/ta-datatable.service.ts`

```typescript
import { Injectable, Logger } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import { PrismaService } from '../../shared/prisma/prisma.service';
import axios from 'axios';
import * as FormData from 'form-data';

@Injectable()
export class TaDatatableService {
  private readonly logger = new Logger(TaDatatableService.name);
  private readonly apiHost: string;
  private readonly token: string;
  private readonly projectId: string;
  private readonly datatableName: string;

  constructor(
    private readonly configService: ConfigService,
    private readonly prisma: PrismaService,
  ) {
    this.apiHost = this.configService.get<string>('TA_API_HOST') || '';
    this.token = this.configService.get<string>('TA_PROJECT_TOKEN') || '';
    this.projectId = this.configService.get<string>('TA_PROJECT_ID') || '22';
    this.datatableName =
      this.configService.get<string>('TA_DATATABLE_NAME') || 'cps_group';
  }

  /**
   * 审批通过后，将角色信息回写到数数平台 CPS 维度表。
   * 自包含方法：内部完成 accountId 查询、CSV 生成、API 调用、日志记录。
   * 调用方只需 fire-and-forget，无需处理异常。
   */
  async writeCpsDim(roleId: string, cpsGroup: string): Promise<void> {
    if (!cpsGroup) {
      this.logger.warn(`跳过CPS维度表回写: cpsGroup为空, roleId=${roleId}`);
      return;
    }

    if (!this.token || !this.apiHost) {
      this.logger.warn('跳过CPS维度表回写: TA_API_HOST 或 TA_PROJECT_TOKEN 未配置');
      return;
    }

    try {
      const role = await this.prisma.role.findUnique({
        where: { roleId },
        select: { accountId: true },
      });

      if (!role?.accountId) {
        this.logger.warn(
          `跳过CPS维度表回写: 角色 ${roleId} 无 accountId`,
        );
        return;
      }

      // CSV 必须包含全部 3 列，首列 account_id 为主键
      const csvContent = [
        'account_id,role_id,cps_group',
        `${this.escapeCsvField(role.accountId)},${this.escapeCsvField(roleId)},${this.escapeCsvField(cpsGroup)}`,
      ].join('\n');

      const form = new FormData();
      form.append('file', Buffer.from(csvContent, 'utf-8'), {
        filename: 'cps_dim_update.csv',
        contentType: 'text/csv',
      });

      const url = `${this.apiHost}/open/datatable/updateDatatable`;
      const response = await axios.post(url, form, {
        params: {
          token: this.token,
          projectId: this.projectId,
          datatableName: this.datatableName,
          updateType: 'INCR_UPDATE',
        },
        headers: form.getHeaders(),
        timeout: 30000,
      });

      const returnCode = response.data?.return_code;
      const succeeded = response.data?.data?.successRowCount ?? 0;
      const failed = response.data?.data?.failedRowCount ?? 0;

      if (returnCode === 0 && succeeded > 0) {
        this.logger.log(
          `CPS维度表回写成功: accountId=${role.accountId}, roleId=${roleId}, cpsGroup=${cpsGroup}`,
        );
      } else {
        this.logger.warn(
          `CPS维度表回写异常结果: returnCode=${returnCode}, succeeded=${succeeded}, failed=${failed}, ` +
          `response=${JSON.stringify(response.data)}`,
        );
      }
    } catch (error) {
      const msg = error instanceof Error ? error.message : String(error);
      this.logger.error(
        `CPS维度表回写异常: roleId=${roleId}, cpsGroup=${cpsGroup}, error=${msg}`,
      );
    }
  }

  private escapeCsvField(value: string): string {
    if (value.includes(',') || value.includes('"') || value.includes('\n')) {
      return `"${value.replace(/"/g, '""')}"`;
    }
    return value;
  }
}
```

## 9. 错误处理与异常场景

| 场景 | 处理方式 | 影响范围 |
|------|---------|---------|
| `TA_API_HOST` 或 `TA_PROJECT_TOKEN` 未配置 | warn 日志，跳过回写 | 仅回写不执行 |
| 角色无 `accountId` | warn 日志，跳过回写 | 仅回写不执行 |
| `apply.platform` 为空 | warn 日志，跳过回写 | 仅回写不执行 |
| 数数平台 HTTP 4xx | error 日志（含 response body） | 仅回写失败 |
| 数数平台 HTTP 5xx / 超时 | error 日志（含 error message） | 仅回写失败 |
| `return_code != 0` 或 `successRowCount=0` | warn 日志（含完整 response） | 回写未生效，需排查 |
| CSV 值含逗号/引号/换行 | `escapeCsvField` 自动转义 | 不受影响 |

**补偿机制（二期考虑）：** 如上线后回写失败率 > 5%，可增加失败记录表 + 定时重试任务。一期仅做日志记录。
